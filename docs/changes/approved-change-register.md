# Approved Change Register

This document records changes that have explicit human approval. Proposed or discussion-state changes do not belong here; they live only under `docs/changes/proposals/` until approval is explicit.

## Register Rules

- add a row only after explicit human approval
- keep the proposal artifact path current
- keep lifecycle state current from `approved` onward
- do not treat this register as a substitute for the parity ledger
- approved changes must still be rowized by the Architect before execution begins
- `next owner` must reflect whether the change belongs to Discussion, Architect, or linked execution rows
- `resulting slice_ids` must be updated as soon as rowization happens
- `closure status` must reflect whether the linked execution rows are still in flight or fully `verified complete`
