# Parity Status Ledger

This document is the canonical live execution state for repo-completion work. `contract-inventory.md` defines family order and census source material; this ledger defines the exact rows, their current state, and their next owner.

## Ledger Rules

- A row is the atomic unit of repo completion.
- `slice_id` is stable and maps to `docs/migration/handoffs/<slice-id>/`.
- `status` must follow the repo-completion protocol pipeline.
- `owner` is the next role responsible for moving the row forward.
- Rows sourced from approved changes must record the originating `change_id` in the note field.
- Rows naming multiple controllers, multiple workflows joined by `and`, or more than one primary behavior are invalid and must be split.
- The repo is complete only when every row below is `verified complete`.

## Status Pipeline

`unplanned` -> `architecting` -> `architected` -> `scaffolding` -> `scaffolded` -> `coding` -> `implemented pending verification` -> `verified complete`

Available non-terminal blocked states:

- `blocked on environment`
- `blocked on contract`
- `blocked on conflicting workspace state`
